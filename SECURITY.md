# Security Policy

## Supported versions

This project is in active development. Security fixes are applied to the `main` branch.

## Reporting a vulnerability

Please do not open public issues for suspected vulnerabilities.

Instead:

1. Use GitHub private vulnerability reporting if available.
2. If unavailable, contact the repository owner directly via GitHub profile.
3. Include reproduction steps, impact, and affected files/versions.

We will acknowledge reports as quickly as possible and work on a fix.
